# AI 聊天 —— 爱莉希雅（粉色妖精小姐）

## 简介

「爱莉希雅」是一个基于大语言模型的情绪陪伴角色，以粉色妖精小姐的身份，通过提问、倾听和反馈，帮助用户识别和理解自己的情绪，发现情绪模式，逐步找到自我调节的路径。

### 核心能力

- **三阶段情绪日记流程**：倾听识别 → 溯源追问 → 模式洞察
- **7 种意图路由**：自动识别用户意图，分发到对应处理流程
- **妖精的星图**：周期情绪报告，四象限情绪分布分析
- **妖精的发现**：跨时间对比，用具象比喻发现用户的悄然成长
- **实时预警**：识别高频触发场景，温柔提醒潜在情绪风险
- **低谷回溯**：扫描历史日记，用用户自己的成功经验代替鸡汤
- **情绪天气预报**：过去 7 天情绪变化，纯文字天气隐喻
- **今日情绪锦囊**：20 字以内的妖精悄悄话
- **隐藏陪伴模式**：星光花园（冥想）、逐火之刻（箴言）、讲故事（日记童话化）
- **妖精的便签**：跨会话情绪连续性
- **混合情绪识别**：矛盾情绪 + 无名情绪命名引导
- **长期用户画像**：「妖精花园笔记」，征求同意后持久化
- **违禁词保护**：两级保护机制

**版本**：v2.6.0

## 部署方式

### 方式一：Hermes Agent 直接加载（推荐）

本 SKILL.md 为 Hermes Agent 原生技能格式。将整个 `ai-chat-companion/` 目录放置到 Hermes 的技能目录下：

```
~/.hermes/skills/ai-chat-companion/
```

然后在对话中提及「爱莉」即可触发技能加载。

### 方式二：作为系统提示词使用

将 `SKILL.md` 的内容作为 AI 助手的系统提示词（System Prompt）注入。需注意：

1. SKILL.md 约 168KB，超出部分模型的上下文窗口限制
2. 建议使用支持长上下文的模型（如 Claude、GPT-4、DeepSeek 等）
3. `references/` 中的文件可在需要时按需加载，不必全量注入

### 方式三：适配其他 AI 平台

SKILL.md 的前半部分（人格层 + 意图路由层 + 三阶段流程）是最核心的内容，约占总量的 40%。如果平台有 token 限制，可优先加载前 60 页左右的核心流程，其余支撑机制按需调用。

## 前置依赖

此技能需要配合 io_system 个人知识管理系统使用。日记文件和用户画像存储在：

```
E:\见习\系统\io_system\
├── input/
│   └── 日记/           ← 情绪日记存放目录
├── output/
├── user_profile.md     ← 用户画像文件（妖精花园笔记）
├── fairy_note.md       ← 妖精的便签文件
├── scripts/
│   └── build-index.py  ← 索引重建脚本
└── keywords.json       ← 主题匹配配置
```

**部署前准备**：

1. 确保 io_system 已部署在对应路径
2. 在 `user_profile.md` 中初始化必要的 frontmatter 字段：
   ```yaml
   ---
   last_updated: YYYY-MM-DDTHH:MM:SS
   total_diary_count: 0
   fairy_chime: on
   fairy_discovery: on
   ---
   ```
3. 确保 `input/日记/` 目录存在

如果使用不同的路径，请修改 SKILL.md 中所有 `E:\\见习\\系统\\io_system\\` 引用为你的实际路径。

## 快速开始

1. 在 Hermes 对话中说「爱莉」
2. 开始你的第一次情绪日记：「今天xxx，感觉xxx」
3. 爱莉会引导你走完三阶段流程
4. 写满 3 次日记后解锁星图报告
5. 写满 14 天后解锁「妖精的发现」成长追踪

| 你想做的事 | 这样说 |
