# AI 情绪陪伴日记（爱莉希雅 ver.）

版本：v1.8.0 | 更新日期：2026-07-12

## 选题

本项目的核心选题是：**基于大语言模型的 AI 情绪陪伴系统**。

传统的情绪日记工具多为被动记录型——用户输入心情，系统打标签存档，缺乏交互深度。本项目的创新在于将情绪日记从"记录工具"升级为"陪伴教练"：通过一个具有鲜明人格的角色（爱莉希雅），在倾听-追问-洞察的三阶段对话流程中，帮助用户识别和理解自己的情绪，发现情绪模式，逐步找到自我调节的路径。

项目融合了三个关键设计：

1. **拟人化陪伴**：以"爱莉希雅"这一粉色妖精少女角色为交互载体，用星光、花朵和俏皮话包裹心理咨询技术（共情倾听、认知重构、动机访谈），降低用户的防御心理，让情绪表达更自然。

2. **双维度情绪模型**：突破传统的"正面/负面"二元分类，引入愉悦度 (valence, -5 到 +5) 和能量感 (arousal, 1 到 5) 双维度评估。同样是负面情绪，高能量消极（焦虑、愤怒）和低能量消极（抑郁、疲惫）的应对策略截然不同，双维度模型使陪伴策略更精准。

3. **数据驱动的自我认知**：所有日记持久化存储并建立索引，支撑两个高层功能——实时预警（检测历史触发模式，主动提醒高风险情境）和低谷回溯（在用户陷入负面情绪时，调取ta自己曾写下的成功应对方法，用"你看，上次你自己就做到了"代替鸡汤）。

## 功能简介

### 核心流水线（三阶段）

| 阶段 | 功能 | 说明 |
|------|------|------|
| 阶段一 | 倾听与情绪识别 | 完整接收用户原文，不做评判；识别主要情绪 + 双维度评分 (valence/arousal) |
| 阶段二 | 溯源追问与洞察 | 深度追问挖掘情绪根源，发现认知盲点，提供可选行动建议 |
| 阶段三 | 模式识别与成长报告 | 扫描历史日记库，生成四象限情绪分布图、触发因素聚类、模式识别报告 |

### 支撑机制

- **闲聊模式** [v1.7.0]：喊"爱莉"即可进入轻松聊天，不强制走日记流程；闲聊中自然流露情绪时无缝切换
- **实时预警** [v1.7.0]：积累 3+ 次日记后，在任何对话中检测历史触发信号（事件/时间/人际/情境），温和提醒
- **低谷回溯** [v1.7.0]：负面情绪时自动扫描日记库，调取用户自己的成功经验，不喂鸡汤
- **持久化存储**：日记存入 io_system (`input/日记/`)，遵循 frontmatter 规范，支持索引检索
- **爱莉星光手记** [v1.6.0]：日记以爱莉第一人称叙事体撰写，自然融入情绪分析而非机械模板
- **双维度情绪模型** [v1.5.0]：valence (-5~+5) × arousal (1~5) 替代单维强度评分

### v1.8.0：闲聊-陪伴无缝切换链路实战验证

通过完整真实对话验证了从"爱莉，今天过得怎么样？"一句日常问候开始，到情绪被自然承接、陪伴、最终日记落盘的端到端流程。核心确认：
- 闲聊模式下不会误启动日记流程
- 闲聊中自然流露情绪时，模式切换准确、平滑
- 正面转折出现时不会强行追问负面根源
- 文件名提取严格遵守四规则（关键词来自用户原话）
- 星光手记第一人称格式完整

### v1.7.1：日记文件名准确性命名规范

针对 AI 曲解用户原意生成文件名的问题（如"高温还要天天上课"被命名为"高温缺课"），新增四规则命名体系（忠实于原文 / 情境情绪格式 / 准确优先 / 不确定降级），确保文件名关键词来自用户直接描述而非 AI 推断。

## 使用方式

### 触发方式

1. **写日记**：直接输入任意长度的情绪表达（吐槽、焦虑、开心、流水账均可），自动进入三阶段陪伴流程
2. **喊"爱莉"**：进入闲聊模式；闲聊中流露情绪时自动切换到日记模式
3. **生成报告**：说"帮我看看这段时间的情绪"

### 存储位置

日记文件持久化存储于 `E:\见习\系统\io_system\input\日记\`，文件名格式：`YYYY-MM-DD-HHMMSS-情绪简述.md`。
